Architecture

ITCSS

ITCSS is a project architecture designed by Harry Roberts. The goal is to ensure a good maintenance and scalabitily of the codebase, by following some conventions and rules.

ITCSS means Inverted Triangle CSS, and represent the complexity graph how your code must be organired.

The triangle represents the CSS selectors order : from generic to explicit, from low-complexity to high-specificity or from global zones to localized ones.

Glsass

In Glsass, the ITCSS is slightly modified, to allow more flexibilty and because of Sass Module. Since we don't have to import all file at the root file, there is not need for you to import in your index.scss.

This is the layer structure:

  • Core (Settings + Tools)
  • Generic
  • Vendors
  • Elements
  • Objects
  • Layouts
  • Components
  • Patterns
  • Scopes
  • Themes
  • Utilities

Core

The Core layer can is combination of the Settings and Tools folders.
The settings contains variables and definitions used in the framework. The tools is about Sass mixins and functions.

Generic

This is the first layer which will generate "real" CSS. It contains reset and normalize styles, box-sizing definitions and some shared styles (like default margin-bottom on some elements)

Elements

Also called Base, there is rules for styling basic HTML elements (heading, links, lists, etc.) to adjust some default browser behavior/feeling.

Objects - .o-*

The first class-based selectors start here, it defines undecorated (or at least, as much as possible) design patterns. Some objects like o-media are known from OOCSS and present in this layer.

Layouts - .l-*

Glsass doesn't provide layout classes because it will depend of your HTML structure. You can add your own folder and files if you use this layer logic (header, main, nav, footer, etc.).

In some cases, you can also have a pages folder to manage some page behavior. But you may avoid to use the .p-* class prefix because it's used for another layer. If you need it, you can use the .page-* prefix (.page-home, .page-about, etc.)

Components - .c-*

In this layer, you have specific UI "components" (or UI blocks), so the majority of the work will go here. A component is a well-designed piece of UI that can be reused in different needs and areas. Glsass provide basic and customizable acomponents (alert, button, card, etc.), that you can use or not, modify them from the Sass parameters or build your own.

Patterns - .p-*

Eventually called Modules (.m-*), a pattern is very similar to a component, because it's an UI block. But if a component is create to be reusable anywhere, a pattern should be built for a unique need.
By convention, a component can be uses a lot of times, in different areas or contexts. For a pattern, it's to create a block for a specific use. The goal is just to organize your UI folder and indicate to the others developers, which ones is reusable and which are not.
Since this layer is specific of the design to build, Glsass didn't contain any pattern. But it can be use to add a new behavior to an existing (external) component (ex: the top-right button of this website to enable the debug mode).

Scopes - .s-*

Scoped context in CSS should be used sparingly. The goal is to solve very specific and local problem. The perfect example is when you use a CMS with a WYSIWYG editor. You can add a class like .s-cms-content around the generated content, and then target HTML elements (ul/li, a, hn).
Before use it, be sure that you cannot achieve the expected result otherwise. And be careful about the weight/priorities of the new classes.

Themes - .t-*

This layer is similar to the previous one, because it must be used only for specific cases, especially if you work with Stateful Themes. But since with the Sass syntax you can use the parent selector &, theming can be applied in components directly.
The theming can also be used from the settings variables, so this layer shouldn't be used ofter.

Utilities - .u-*

Trumps, helpers or utilitities, this is the last layer of your stack.
Glsass already contains a lot of utilities classes, mainly inherit from Bootstrap, Tachyon, Tailwind or other CSS frameworks.

Because it can generate a lot of CSS rules, there is not index.scss file inside the folder, you must import and select what you need.
In addition, you can use a tool like PurgeCSS to remove unused classes in your templates.
