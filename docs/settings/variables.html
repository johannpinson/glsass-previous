Variables

The variables file add custom properties (or CSS variables).
It can be usefull if you want to manage some colors modifications from JavaScript by modify the properties (or override them in an inline style tag). 

:root {
  --color-primary: #{cv("primary")};
  --color-primary-rgb: #{red(cv("primary")) + ", " + green(cv("primary")) + ", " + blue(cv("primary"))};
  --color-secondary: #{cv("secondary")};
  --color-secondary-rgb: #{red(cv("secondary")) + ", " + green(cv("secondary")) + ", " + blue(cv("secondary"))};
}

Since this support is limited if you need an IE compatibility, using CSS variables is restricted to specific cases.
That's why it's disabled by default, but you can change it by editing with the $g-enable-variables variable.

// With global settings import
@use "~glsass/src/settings" with ($g-enable-variables: true);

// Or from the direct file import
@use "~glsass/src/settings/settings.variables" with ($g-enable-variables: true);

In next versions of Glsass, more variables will be available (with @mixins to generate others more easily).
